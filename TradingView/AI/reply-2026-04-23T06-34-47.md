## 1. Intrabar Alert Frequency (TradingView Side)

Your alerts can fire **every tick** rather than on bar close:

```
Alert Settings:
├── Frequency: "Once Per Close" ❌ (misses the move)
└── Frequency: "Every Time" ✅ (catches intrabar wicks)
```

**Multi-condition intrabar alert example:**
```python
// Pine Script for ML signal + price condition
if (ml_prediction == "long" and low <= support_level * 1.002)
    alert("ML Long Trigger - Price swept support", alert.freq_once_per_bar_close)
```

**BUT** — "Every Time" can spam alerts during choppy conditions. Use with cooldown:
```python
var int last_alert_time = 0
if (time - last_alert_time > 5 * 60 * 1000)  // 5 min cooldown
    if (condition)
        alert("Signal")
        last_alert_time := time
```

## 2. The "Scout" Timeframe Strategy

Don't trade 30m — use it for **context**, trade the **5m or 1m** for execution:

```
Higher TF (30m): ML prediction + structure context
                 ↓
Lower TF (5m):   Entry execution when price aligns with 30m bias
```

**Workflow:**
1. 30m ML model generates **bias** (long/short/neutral) at bar open
2. 5m model or price-action rules handle **execution** within the 30m bar
3. Entry only if 5m shows the sweep-and-reclaim pattern you want

**Benefit:** You catch the $241 wick on 5m while 30m bar still shows $243 close.

## 4. Intrabar Feature Stability (ML Critical Issue)

Your 120 SHAP features — **which ones are stable intrabar?**

| Feature Type | Intrabar Stability | Risk |
|-------------|-------------------|------|
| EMAs (9/21/50) | Medium | Lag behind fast moves |
| RSI | High | Smooth, less whipsaw |
| Volume | High (with caution) | Can spike on noise |
| Pivot levels | **Perfect** | Fixed for the day |
| Fibonacci | **Perfect** | Based on prior swing |
| Order blocks | **Perfect** | Historical levels |

**Recommendation:** 
- Use **static features** (pivots, Fibs, OBs) for intrabar triggers — they don't change mid-bar
- Use **dynamic features** (RSI, EMAs) for confirmation only
- Recalculate ML prediction **every 1m** but weight static features higher

## 5. Partial Position Scaling (Risk Management)

Don't go all-in on an intrabar signal — **scale in**:

```
First 50%: Intrabar trigger (the $241 wick)
    ↓
Second 50%: 30m bar close confirmation (price holds $243)
    ↓
Full position: Only if both conditions met
```

**If the wick recovers and holds → you caught the bottom with half size**
**If the wick keeps bleeding → you stopped out on half, avoided full loss**
