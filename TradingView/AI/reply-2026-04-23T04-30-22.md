# Pivots & Fibonacci as ML Features

**Yes, pivots and Fibonacci can work well** — but only with proper feature engineering. Raw price levels will fail; you need **relative, normalized encodings**.

---

## The Problem with Raw Levels

| Issue | Why ML Fails |
|-------|--------------|
| Absolute prices ($42,500) | Model memorizes training range, breaks when BTC moves to $60k |
| Static "above/below" binary | Loses distance information — being $10 above S1 vs $100 above is very different |
| Fixed lookback swings | Same "0.618 retracement" at $30k BTC vs $100k BTC has different market structure |

---

## Proper Feature Engineering

### 1. Normalized Position Features (Essential)

```python
# Position within daily pivot range (0 = at S1, 1 = at R1)
pivot_position_daily = (close - daily_s1) / (daily_r1 - daily_s1)

# Distance from pivot in volatility units (ATR-normalized)
pivot_distance_atr = (close - daily_pivot) / ATR(14)

# Position within Fibonacci retracement (0 = swing low, 1 = swing high)
fib_retracement_pct = (close - swing_low) / (swing_high - swing_low)
```

**Why this works:** A value of `0.65` means the same thing whether BTC is at $30k or $100k — it's "65% of the way through the current range."

### 2. Your Multi-Timeframe Pivot Approach

Your instinct is correct. Here's the proper hierarchy:

| Primary Chart | Pivot Source | Usage |
|---------------|--------------|-------|
| 30m | Daily Pivot | Intraday support/resistance |
| 30m | Weekly Pivot | Major structural levels |
| 30m | Monthly Pivot | Macro context |

**Feature encoding:**
```python
# Relative position on each timeframe
daily_pivot_pos = (close - daily_s1) / (daily_r1 - daily_s1)
weekly_pivot_pos = (close - weekly_s1) / (weekly_r1 - weekly_s1)
monthly_pivot_pos = (close - monthly_s1) / (monthly_r1 - monthly_s1)

# Confluence detection
pivot_cluster = 1 if abs(daily_pivot - weekly_fib_618) < ATR(5) else 0
```

### 3. Fibonacci-Specific Features

```python
# Golden pocket flag (highest-probability reversal zone)
in_golden_pocket = 1 if 0.618 <= fib_retracement_pct <= 0.65 else 0

# Distance to nearest significant level
nearest_fib_dist = min(
    abs(fib_retracement_pct - 0.382),
    abs(fib_retracement_pct - 0.5),
    abs(fib_retracement_pct - 0.618),
    abs(fib_retracement_pct - 0.786)
)

# How extended is current move? (for targeting)
extension_to_1272 = (close - swing_low) / (fib_1272_extension - swing_low)
```

## Recommended 10-Feature Pivot/Fib Set

```python
# === PIVOT FEATURES (4) ===
pivot_pos_daily       # Position within daily S1-R1 range
pivot_pos_weekly      # Position within weekly S1-R1 range  
pivot_confluence      # Daily & weekly pivots aligned (0/1)

# === FIBONACCI FEATURES (6) ===
fib_retracement       # 0-1 scale (0=swing low, 1=swing high)

fib_distance          # Distance to nearest key level
fib_touches_382       # Times 0.382 level was tested (20-bar lookback)
fib_touches_618       # Times 0.618 level was tested
extension_progress    # How far to 1.272 target (for take-profit logic)
```

---

## Critical: Avoid Look-Ahead Bias

**Wrong** (leaks future info):
```python
swing_high = max(high[last_20_bars])  # Includes current bar!
fib_618 = calc_fib(swing_high, swing_low)
```

**Right** (clean):
```python
swing_high = max(high[last_20_bars].shift(1))  # Past only
fib_618 = calc_fib(swing_high, swing_low)
```

## Validation: Check Your SHAP Values

After training, your top features should look like:
1. `fib_retracement` (around 0.618 = high importance)
2. `pivot_pos_daily` (extremes = mean reversion signal)
3. `ADX` or trend strength
4. `volume_ratio`

If random features rank higher, you have **data leakage** or **collinearity** (pivot levels too correlated with price/EMAs).
